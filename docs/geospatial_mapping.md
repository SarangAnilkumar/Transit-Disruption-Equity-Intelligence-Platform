# Geospatial Mapping (Milestone 3)

## Purpose
Milestone 3 creates a local-first geospatial bridge between GTFS transport entities and ABS SA2 statistical areas. This enables later equity analysis by area without introducing AWS/warehouse dependencies yet.

## Required input files
- GTFS static processed files (generated by `fetch_gtfs_static.py`):
  - `stops.txt`
  - `routes.txt`
  - `trips.txt`
  - `stop_times.txt`
- ABS SA2 boundary file (GeoPackage, Shapefile, or GeoJSON), e.g.:
  - `data/raw/geospatial/sa2_boundaries/<your_sa2_boundary_file.gpkg>`
- Optional SEIFA file (CSV/XLSX):
  - `data/raw/equity/seifa/<your_seifa_sa2_file.xlsx>`

## Script commands
```bash
python ingestion/ingest_sa2_boundaries.py
python ingestion/build_stop_sa2_mapping.py
python ingestion/build_route_sa2_coverage.py
python ingestion/prepare_seifa_sa2.py
python ingestion/build_sa2_disruption_base.py
pytest -q
```

## Expected outputs
- `data/processed/geospatial/stops_sa2_mapping.csv`
- `data/processed/geospatial/stop_area_mapping.csv` (roadmap alias of the same mapping table)
- `data/processed/geospatial/geospatial_mapping_report.json`
- `data/processed/geospatial/route_sa2_coverage.csv`
- `data/processed/geospatial/route_sa2_coverage_report.json`
- `data/processed/equity/seifa_sa2_ready.csv` (if SEIFA input exists)
- `data/processed/equity/seifa_preparation_report.json` (if SEIFA input exists)
- `data/processed/analytics/sa2_disruption_observations_base.csv` (if parsed trip updates exist)
- `data/processed/analytics/sa2_disruption_observations_report.json`

## Output schema notes
### `stops_sa2_mapping.csv`
- `stop_id`, `stop_name`, `stop_lat`, `stop_lon`
- `sa2_code`, `sa2_name`
- `mapping_method`, `is_matched`
- `mapping_created_at`, `source_stops_path`, `source_sa2_path`

### `route_sa2_coverage.csv`
- `route_id`, `route_short_name`, `route_long_name`, `route_type`
- `sa2_code`, `sa2_name`
- `stop_count_in_sa2`, `trip_count_serving_sa2`
- `first_stop_sequence`, `last_stop_sequence`
- `coverage_created_at`

### `sa2_disruption_observations_base.csv`
- `snapshot_date`, `snapshot_hour`, `sa2_code`, `sa2_name`
- `observation_count`, `distinct_trip_count`, `distinct_stop_count`
- `delayed_observation_count`
- `avg_arrival_delay_seconds`, `avg_departure_delay_seconds`
- `max_arrival_delay_seconds`, `max_departure_delay_seconds`
- `unmatched_stop_observation_count`

## QA checks
- Stop coordinate validity checks (`lat/lon` numeric and in range).
- SA2 column detection from common ABS naming variants.
- Mapping match-rate warning when below 90%.
- Route/SA2 aggregation validated with unit tests.
- Disruption aggregation validated with unit tests.

## Known limitations
- Stop coordinates can fall outside polygon boundaries due to geometry precision or feed quality.
- SA2s are statistical boundaries, not true transport catchments.
- A stop mapped to SA2 does not imply all residents in that SA2 use that stop.
- Route-to-SA2 coverage is a network exposure proxy, not ridership.
- GTFS-R observations are snapshots, not complete official reliability records.
